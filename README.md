## AI가 설계한 주요 테스트 시나리오
실제 쇼핑몰의 유저 저니(User Journey)를 AI가 스스로 추론하여 설계한 6가지 핵심 E2E 테스트 케이스입니다. (e2e/ 폴더 참조)

- 인증 (auth.spec.ts): 회원가입, 로그인, 로그아웃 검증
- 상품 목록 및 탐색 (products.spec.ts): 상품 목록 렌더링, 필터링, 검색 기능 검증
- 상품 상세 (product_detail.spec.ts): 상품 상세 정보 확인 및 옵션 선택 검증
- 장바구니 (cart.spec.ts): 장바구니 담기, 수량 변경, 삭제 검증
- 결제 (checkout.spec.ts): 배송 정보 입력 및 주문 결제 완료 흐름 검증
- 엣지 케이스 (edge_cases.spec.ts): 비정상적인 입력 및 네트워크 지연에 대한 방어 로직 검증

## CI/CD 자동화 (GitHub Actions)
AI가 스스로 판단하여 로컬 테스트를 넘어선 지속적 통합(CI) 파이프라인까지 구축했습니다.

- 자동 실행 트리거: main 브랜치에 코드가 푸시되거나 풀 리퀘스트(PR)가 생성될 때마다 E2E 테스트가 자동으로 구동됩니다.
- 테스트 리포트 확인: 테스트 완료 시 GitHub 저장소의 Actions 탭 하단 Artifacts 항목에 Playwright HTML 리포트가 업로드되도록 AI가 workflow를 작성했습니다.
